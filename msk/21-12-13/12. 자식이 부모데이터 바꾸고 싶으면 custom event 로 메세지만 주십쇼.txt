12. 자식이 부모데이터 바꾸고 싶으면 custom event 로 메세지만 주십쇼

    - props로 보낸 데이터는 Read-only (수정금지)

    - 자식 컴포넌트가 부모 컴포넌트에 있는 데이터를 수정하고 싶을 때 Custom Event 사용

    - 이벤트 버블링 : 자식 태그를 눌렀을 때 상위 태그에도 전부 클릭 이벤트 발생
        - 이벤트 버블링을 활용하여 부모 컴포넌트 쪽에서 자식 컴포넌트 태그에 @click="함수명" 형태로 이벤트를 걸어둘 수도 있음
        - 예) App.vue 내에서 아래와 같이 Card 컴포넌트 작성
            - <Card @click="클릭이벤트"></Card>

    - 자식 컴포넌트에서 부모 컴포넌트로 보내기
        - $emit('변수명', 데이터) : 부모에게 메시지를 보내는 기능
        - $ 표시는 Vue만의 특별한 변수 (jquery처럼)
        - script 쪽에 함수를 빼서 작성할 때는 this.$emit() 형태로 사용해야 함

    - 부모 컴포넌트에서 자식 컴포넌트 메시지 수신하기
        - <자식컴포넌트 @변수명="실행할 함수"></자식컴포넌트>
        - 자식 컴포넌트에서 $emit에 입력한 변수명을 사용
        - 자식이 보낸 데이터는 $event 변수에 담겨 있음
